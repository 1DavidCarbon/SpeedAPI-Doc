Get Leaderboard

Description

Retrieve the leaderboards.

ATT: although technically this is a get, this endpoint is implemented as a http POST call.
The reason for this is the 2k limitation on http GET calls, which is not sufficient when querying with a long list of driver names.

URI

POST /ws/game/nfsw/leaderboards

Input Parameters
FieldType Required? Format Description
output

 

 Optional

 

Format of response, either json or xml. The default format is xml.
eid

 

 Required

 

Event Id.
et

 

 Required

 

Type of the event. Valid Event Types are: PVP = 1, PVE = 2.
lt

 

 Required

 

Type of the leaderboard. Valid Leaderboard Types are: Overall = 1, Daily = 2, Weekly = 3, Monthly = 4.
dn

 

 Required

 

A comma-seperated list of driver names to fetch the leaderboard for.
rf

 

 Optional

 

The record filter. It's used with Daily, Weekly and Monthly leaderboard. Daily leaderboards (0-6): Days past from current day, while 0 means current day and 6 means 6 days before. Weekly leaderboards: A valid week number using standard US English culture rules (the first day of the week of a given year is Sunday). Monthly leaderboards (1-12): A valid month number.
shard

 

 Optional

 

Retrieve information from a specific shard. The default is the configured default shard in server configuration. An example would be "Apex".
